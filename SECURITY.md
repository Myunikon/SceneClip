# Security Policy

## Supported Versions

Currently, only the latest release of SceneClip is supported with security updates. We highly recommend all users to stay updated with the latest version.

| Version | Supported          |
| ------- | ------------------ |
| >= 1.5.0| :white_check_mark: |
| < 1.5.0 | :x:                |

## Reporting a Vulnerability

Security is a top priority for SceneClip. If you discover a security vulnerability within SceneClip, please do NOT report it by creating a public GitHub issue.

Instead, please send an e-mail directly to the project maintainer or create a private GitHub security advisory.

Please include the following information in your report:

- Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We will evaluate the report and respond as soon as possible. We will keep you informed of our progress towards a fix and announcement, and may ask for additional information or guidance.

We appreciate all efforts to discover and coordinate the disclosure of security vulnerabilities. We will properly acknowledge your contribution.
